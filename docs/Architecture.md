## 3. External Service Integration Architecture

### Google Places API (New)
```
Client вҶ’ Discovery Service вҶ’ GoogleTextSearchClient
  вҶ’ POST places:searchText   (Field mask for cost control)
  вҶ’ Response parsed into DiscoveryPlaceResult
  вҶ’ Cached in Redis (1 hour)
  вҶ’ Audited to search_queries + search_results
```

### Google Routes API
```
Client вҶ’ Routes Service вҶ’ GoogleRoutesClient
  вҶ’ POST computeRoutes / computeRouteMatrix
  вҶ’ RouteResult parsed with traffic delay computation
  вҶ’ Cached in Redis (5 min routes, 2 min matrix)
```

### OpenAI (Embeddings + Chat)
```
Client вҶ’ Knowledge Service вҶ’ OpenAIEmbeddingClient
  вҶ’ POST embeddings (batch, max 100 texts)
  вҶ’ Extracted vectors stored in Pinecone
  
Client вҶ’ Place Q&A Service вҶ’ OpenAIEmbeddingClient
  вҶ’ POST chat/completions (streaming for WebSocket)
  вҶ’ RAG context built from Pinecone matches + structured DB data
```

### Pinecone (Vector Database)
```
Knowledge Sync вҶ’ Chunk place data вҶ’ Embed вҶ’ Upsert vectors
Place Q&A вҶ’ Embed question вҶ’ Query Pinecone вҶ’ Filter by score
  вҶ’ Build context вҶ’ Call OpenAI вҶ’ Return grounded answer
```

## 5. WebSocket Protocol

### Connection Flow
```
1. Client вҶ’ WebSocket connect to /ws/chat
2. Server accepts connection
3. Client sends auth message: {"type":"auth","token":"Bearer <jwt>"}
4. Server validates вҶ’ sends {"type":"connected","user_id":...}
5. Client sends messages (see below)
6. Server streams token-by-token responses
```

### Message Types
| Type | Direction | Purpose |
|------|-----------|---------|
